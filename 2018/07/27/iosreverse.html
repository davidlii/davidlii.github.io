iOS 应用逆向工程

2018-07-27

1、简介

软件的逆向工程，指的是通过分析一个程序或系统的功能、结构或行为，将它的技术实现或设计细节推导出来的过程。当我们因为工作需要，或是对一个软件的功能很感兴趣，却又拿不到它的源代码时，往往可以通过逆向工程的手段对它进一步分析。

1.1.作用

iOS 逆向工程是一个在软件层面上进行逆向分析的过程。我们可以通过逆向一些经典的软件，包括竞争对手的软件，看看自己的软件能够借鉴些什么技术，如设计思路、内部算法和实现细节等等。

1.2.过程

一般来说，软件逆向工程可以看作系统分析和代码分析两个阶段的有机结合。在系统分析阶段，我们从整体上观察目标程序的行为特征，文件的组织架构，从而找到我们感兴趣的地方。一旦我们能够较为熟练地操作这个程序，并且找到了我们想要逆向的点，就可以进入代码分析阶段，利用各种工具对程序本身的二进制文件进行分析，达到我们的目的。

1.3.工具

  • 一台越狱设备。

具体的越狱流程可以参考盘古、pp助手等知名越狱软件的官网。

  • 监测工具

在逆向工程中，起到嗅探、监测、记录目标程序行为的工具统称为监测工具。这些工具通常可以记录并显示目标程序的某些操作，如网络活动、文件访问等。iOS 逆向常用的监测工具有 Reveal、 tcpdump、libNotifyWatch，PonyDebugger 等。

  • 开发工具

Xcode 是最常用的开发工具，但在此之外，还有可以用于开发越狱代码的工具，其中最方便也最有名的一个工具就是 Theos。

  • 反汇编器

IDA 是逆向工程中最常用的工具之一，横跨 Windows/Linux/Mac 平台，它把目标程序的可执行文件作为输入，输出这个程序的汇编代码，甚至伪代码。
另一个反汇编利器是 Hopper Disassembler，作用与 IDA 类似，也可以把机器码解析成汇编代码和伪代码。

  • 调试器

在 Xcode 中我们因为有自己的代码，所以可以方便的进行单步调试。而在逆向工程中，我们往往没有目标程序的源代码，所以就需要借助调试器来设置断点、单步调试等。常用的调试器有：gdb、lldb、Cycript。

1.4.目标文件

iOS逆向的目标包括 Application、Dynamic Library、Daemon（守护进程）等二进制文件。这里要讲的是 Application，即我们熟悉的 App。应用又分为两种类型：一种是系统 App，越狱设备中来自 Cydia 的应用也被视为系统 App；另一种则是从 AppStore 下载的应用，简称 Store App。两者有一些差异：

  • 目录结构：

Store App 存放在/var/mobile/Containers/目录下，而系统系统 App 和 Cydia App 则存在/Applications/ 目录下。Store App 的数据目录在/var/mobile/Containers/Data/下。以 mobile 权限运行的系统 App 的数据目录在/var/mobile下，而以 root 权限运行的系统 App 的数据目录在/var/root下。

  • 安装包格式与权限：

Cydia App 的安装包格式一般是 deb，以 root 权限运行； Store App 的安装包格式则为 ipa，只能以 mobile 权限运行。

2、砸壳

从 AppStore 下载的应用是被苹果特殊加密过的，可执行文件被套上了一层保护壳。对这类应用想 dump 出它的头文件等，需要先解密应用的可执行文件，这个过程就被称为砸壳。dumpdecrypted 就是由越狱社区知名人士 Stefan Esser 出品的一款砸壳工具，被越狱社区广泛应用于iOS的逆向工程研究中。
 

dumpdecrypted的工作原理是：将应用运行起来（iOS系统会先解密应用可执行文件再启动），然后遍历 Load Command 中所有 LC_ENCRYPTION_INFO 或 LC_ENCRYPTION_INFO_64的信息，将对应解密后的数据从内存中 dump 出来，复写到 mach-o 文件中，得到一个新的可执行程序。

3、反汇编

接下来就可以对此砸壳后的文件进行静态分析了，如用class-dump导出 App 的头文件、用 Hopper Disassembler查看伪代码。

3.1.class-dump

class-dump 是用来 dump 目标对象的 类、分类、协议信息的工具，这一点与otool类似。它利用 OC 语言的 runtime 特性，将存储在 Mach-O 文件中的头文件信息提取出来，并生成对应的.h文件。在 传送门 下载最新安装包，然后把class-dump文件放到/usr/local/bin目录下， 在终端输入class-dump，显示class-dump的版本后，就可以正常使用 class-dump 命令了。

3.2.Hopper

class-dump 只能查看目标 APP 的头文件，无法查看.m 文件和具体的代码，所以就需要使用到反汇编器。这里介绍的是Hopper Disassembler，它是一款二进制反汇编器，能反编译出二进制文件中的所有函数和实现，包括伪代码以及控制流图(Control Flow Graph)，支持 ARM 指令集并针对 OC 做了优化。官网 传送门，下载试用版就够用了。

选择需要反编译的架构之后点OK，接下来 Hopper 就会自动加载二进制文件的各个segment。加载完成后即可看到如下界面：

左边是符号 Label 等区域，中间是 ARM 的汇编代码，右边是相关信息栏。
