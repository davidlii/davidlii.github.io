事件传递 & 事件响应、hitTest

2018-01-07

1、UIResponder

此类用来响应用户的操作并处理各种事件：

  • 触摸事件
  • 加速计事件
  • 远程控制事件

1.2.触摸事件的产生？

当用户在屏幕上做手指点击、缩放等操作时，就会触发触摸事件。此时 UIKit 会创建一个 UIEvent 事件对象，该对象会被添加到一个事件队列中。后续会由 UIApplication 负责事件的分发和传递，直到找到一个合适的响应视图。

2、触摸事件的传递？

事件的传递过程其实就是点击检测的过程，即查找事件触发者的过程。这个过程中，响应者对象需要做两个判断条件：自己是否能接收事件？触摸点是否在自己身上？

按照响应者的先后顺序，如下传递：

  1. UIApplication 从事件队列中取出最前面的一个事件，传递给主窗口（keyWindow）。
  2. 主窗口：判断自己是否能接收触摸事件？触摸点是否在自己身上？
  3. 假如主窗口满足上面两个条件（即能接收事件且触摸点在自己身上），则按照从后往前的顺序遍历主窗口的子视图数组（…C、B、A）。对这些子视图逐一重复以上两个判断。
  4. 假如子视图中没有符合条件的，则主窗口成为最合适的视图。
  5. 假如主窗口的子视图A符合条件，则继续遍历 A 的子视图，并重复上面的两个判断。
  6. 同样，假如A的子视图中没有符合条件的，则A成为最适合的视图。假如A的子视图中有符合条件的（X），则继续遍历X的子视图，直到找到最合适的视图。

ps：触摸事件的传递是从父视图传递到子视图，如果父视图不能接受触摸事件，那么子视图就不可能接收到触摸事件。

2.1.哪些情况不能接收事件？

  • userInteractionEnabled = NO;
  • hidden = YES; （注：父视图隐藏时，子视图也会隐藏，不能接收事件）
  • alpha < 0.01;

2.2.怎么判断触摸点在自己身上？

返回YES，则触摸点在当前 view 上。

只要事件一传递给一个视图，这个视图就会调用它自己的 hitTest:withEvent: 方法。如果 hitTest:withEvent: 方法中返回nil，那么调用该方法的视图本身和其子视图都不是最合适的 view，也就是在自己身上没有找到更合适的 view。那么最合适的 view 就是该视图的父视图。

找到最合适的视图视图后，就会调用视图的 -touches 系列方法来作具体的事件处理。

2.3.事件的拦截？

正因为 hitTest:withEvent: 方法可以返回最合适的 view，所以，想让谁成为最合适的 view 就重写谁自己父视图的 hitTest:withEvent: 方法返回指定的子视图。或者重写自己的 hitTest:withEvent: 方法 return self。但是，建议在父视图的 hitTest:withEvent: 中返回子视图作为最合适的view！

3、响应者链

上面说到，事件传递的过程其实就是查找事件触发者的过程，检测到了触摸的对象之后，事件到底是如何响应呢？这个过程就必须引入一个新的概念“响应者链”。

在 iOS 程序中无论是最后面的 UIWindow 还是最前面的某个按钮，它们的摆放是有前后关系的。一个视图可以放到另一个视图上面或下面，那么用户点击某个视图时是触发上面的视图还是下面的视图呢？这种先后关系构成一个链条就叫“响应者链”。也可以说，响应者链是由多个响应者对象连接起来的链条。

3.1.事件的响应？

在找到最合适的 view 之后就会调用视图的 -touches 方法，-touches 默认做法是把事件顺着响应者链条向上抛。

  1. 如果当前 view 是控制器的 view，那么控制器就是上一个响应者，事件就传递给控制器；如果当前 view 不是控制器的 view，那么父视图就是当前 view 的上一个响应者，事件就传递给它的父视图。
  2. 在视图层次结构的最顶级视图，如果也不能处理收到的事件或消息，则其将事件或消息传递给 window 对象进行处理。
  3. 如果 window 对象也不处理，则其将事件或消息传递给 UIApplication 对象。
  4. 如果 UIApplication 也不能处理该事件或消息，则将其丢弃。

在事件的响应中，如果某个视图实现了 touches…方法，则这个事件将由该视图来接受，如果调用了 [super touches….];就会将事件顺着响应者链条往上传递，传递给上一个响应者；接着就会调用上一个响应者的 -touches…. 方法。

3.2.事件的传递和响应的区别？

  • 事件的传递是从上到下（父视图到子视图）。
  • 事件的响应是从下到上（顺着响应者链条向上传递：子视图到父视图）。
