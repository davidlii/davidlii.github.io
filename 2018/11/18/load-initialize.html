+load 与 +initialize

2018-11-18

有时，我们需要在应用启动阶段，或者某个类初始化之前处理一些特殊的需求，比如初始化全局变量等。OC 在NSObject 中有如下两个类方法：

+load与+initialize的共同点是，这俩方法在应用的整个生命周期内只会自动调用一次，不建议主动调用这两个方法。

+load

Invoked whenever a class or category is added to the Objective-C runtime; implement this method to perform class-specific behavior upon loading.

+load在 dyld 加载镜像文件 -> 初始化runtime -> 注册OC类与分类之后被调用。

通过这个方法，可以在应用启动阶段，即加载应用可执行文件时、进入程序主入口前，实现一些特殊功能。

调用顺序

Xcode 直接 run 之后，输出日志：

  • 当子类和超类都实现这两个方法时，都会触发，且子类触发时间晚于超类；
  • 当原类和分类都实现这两个方法时，都会触发，且分类触发时间晚于原类；
  • 当有多个类都实现此方法时，各类之间的触发顺序不确定；

+initialize

The runtime sends initialize to each class in a program just before the class, or any class that inherits from it, is sent its first message from within the program. Superclasses receive this message before their subclasses.

+initialize会在每个类或其子类收到第一个消息之前调用，如果类一直没被调用，则其中的这个方法一直不会触发；

调用顺序

调用及日志：

  • 当原类和分类都实现此方法时，只有分类中的方法会触发，因为查询方法列表时先查到的是分类中的 initialize；
  • 当只有超类实现此方法，子类没有实现此方法时，对于每个子类，第一次向其发送消息时，都会触发超类中的 initialize；
  • 当子类和超类都实现此方法，且超类没有分类，或者有分类但分类中没有实现此方法时，首次向超类发送消息，则只会触发超类中的方法；首次向子类发送消息，则两个都会触发且子类晚于超类；先后向子类超类发送消息，则执行到超类时只触发超类的 initialize，执行到子类时只触发子类的 initialize，因为超类中的 initialize 已经触发过了；

比较

  • 从触发的时间序列上来看，+load 发生在二进制文件加载阶段，在 UIApplicationMain 这个程序主入口之前；+initialize 发生在程序完全启动之后，也就是在 +load 之后。
  • load 发生在加载镜像期间，各类的 load 方法加载顺序不确定，所以不建议在一个类中调用另一个类；initialize 在程序启动之后，类均已在运行时环境中创建和加载，所以可以正常调用。
  • load 在启动阶段会全部触发；initialize 只在调用当前类时才会触发；
  • 两个方法在超类、子类和分类等情况下的调用顺序有所不同，注意区分~
