#3.block的类型

#_NSConcreteGlobalBlock

全局的静态 block。不会访问任何外部变量。
 

#_NSConcreteStackBlock

保存在栈中的 block。内部访问了某些外部变量；当函数返回时 block 会销毁。
 

#_NSConcreteMallocBlock

保存在堆中的 block。block 被 copy 时其 isa 指针会被修改为此类型；当引用计数为 0 时 block 会销毁。
 

第#2章节的示例中，__main_block_impl_0 结构体中impl.isa指向的是_NSConcreteStackBlock 类型。这是因为 main.m 中我们在 block 外声明了一个变量int count = 10，并且在 block 内部访问了该变量，所以 block 实例被保存在了栈上。
 

当然，如果我们在 block 内不访问任何外部变量，impl.isa应该是指向 _NSConcreteGlobalBlock 类型，但实际上 clang 编译出来的结果却还是指向 _NSConcreteStackBlock类型：

#4.block 对变量的捕获

第#2章节的示例中，block 内访问了外部变量，第#3章节的示例中未访问外部变量。比较两者编译后的main.cpp文件里构造函数__main_block_impl_0()这部分代码：

可以看到，访问了外部变量时，构造函数__main_block_impl_0()的参数里会多出一个_count，这里就是 对变量的自动捕获。在声明 block 时，栈上的局部变量 count 就已经以参数的形式被传入构造函数中，之后被复制到__main_block_impl_0结构体中。

根据示例，可以得到以下几个信息：

1、全局变量、静态变量、静态全局变量都可以正常在 block 内被重新赋值。而局部变量 anInt 在 block 内部重新赋值时，编译器报错“variable is not assignable”。所以， block 内不能给在它外部声明的局部变量（没有使用__block修饰的情况下）重新赋值。

2、全局变量、静态变量、静态全局变量在初始化时都有默认值，block 内重新赋值后，最终打印出的都是重新修改之后的值。而局部变量 anInt 初始化时 = 0，在调用 block 之前重新赋值为1；调用 block 之后，打印所得信息显示 block 内 anInt 的值仍为赋值前的0。这说明 block 内保留的局部变量是 blok 声明之前的那个值。

3、局部变量 array 赋值时报错“variable is not assignable”，而执行 addObject 操作却不会报错。所以，block 中使用到的局部变量可以执行操作而不能赋值。

4、block 捕获外部变量仅仅只捕获 block 闭包里面会用到的值，其他用不到的变量，它并不会去捕获。

#5.问题来了

为啥全局变量、静态变量在 block 内能修改，而局部变量不能修改？
 
OC的内存区域分为：堆区、栈区、全局区(数据区)、代码区。

其中全局变量、静态全局变量都存储在全局区，所以在 block 内可以直接被修改。

其次，静态变量传递给 block 的是内存地址值，所以能在 block 里面直接改变值。

最后，局部变量是以值传递方式传递到 block 的构造函数里面去的。由于只捕获了局部变量的值，并非内存地址，所以 block 内部不能改变自动变量的值。

想要在 block 中修改变量的值，有2种方式：

  • 传递内存地址指针到 block 中;
  • 改变存储区方式(如__block anInt);
