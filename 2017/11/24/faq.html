2、NSString 的修饰符

copy 修饰的 NSString 类型的属性，在执行 setter 操作时，系统会默认拷贝一份新的对象并把属性的指针指向这个新的对象，新的对象的值跟源对象是一样的，但是二者内存地址不同。所以，后续在源对象上进行的修改，并不会影响到新拷贝的这个对象。
 

而使用 strong 修饰的 NSString 类型的属性，其内存地址与源字符串一样，都指向源字符串在常量区的值。因此当源字符串的值发生变化时，属性会跟着变化。
 

另外，当使用 NSString 类型的源字符串给 NSString 类型的属性赋值时，因为源字符串是不可变的，所以属性不论使用 copy 还是 strong 修饰，都是指向源对象所属的内存地址。

3、block为什么使用copy修饰？

block 是一个对象, 所以 block 理论上是可以retain/release的。但是 block 在创建的时候它的内存是默认是分配在栈(stack)上，而不是堆(heap)上的。 所以它的作用域仅限创建时候的当前上下文(函数, 方法…)，当你在该作用域外调用该 block 时, 程序就会崩溃。
 

一般情况下，你不需要自行调用copy或者retain一个 block。 只有当你需要在 block 定义域以外的地方使用时才需要copy。copy会将 block 从内存栈区移到堆区。
 

其实 block 使用 copy 是MRC留下来的。 在MRC下, 如上述, 在方法中的 block 创建在栈区, 使用 copy 就能把他放到堆区, 这样在作用域外调用该 block 程序就不会崩溃。
 

但在ARC下, 使用 copy 与 strong 其实都一样，因为 block 的 retain 就是用 copy 来实现的。

5、指针&地址

指针变量：存放地址的变量即为指针变量。指针变量是一种特殊的变量，它不同于一般的变量，一般变量存放的是数据本身，而指针变量存放的是数据的地址。
 

例如：int a = 68，系统为变量a分配的首地址为0X065FDF4H。声明一个空指针：long *p = NULL；给其赋值：p=&a。
 

这里a的值为68，&a是指a的地址，即0x065FDF4H。而p就是存放变量a地址的指针变量，即p中存放的是地址0x065FDF4H。

7、静态变量、静态常量

static

修饰局部变量时，静态局部变量的作用范围为该函数体。它的值在编译期就会确定下来，并被存储到全局变量区。因此静态局部变量只会生成一份内存、只会初始化一次并供所有对象使用；静态局部变量的生命周期和程序相同，直到程序结束这个局部变量才会销毁。
 

修饰全局变量时，静态全局变量的作用域仅限于当前文件（.m），能避免重复定义全局变量。
 

静态变量的值是可以更新的，并且只要某个对象对静态变量做了修改，所有的对象都能访问更新后的值。

const

const修饰的是常量，常量在赋值一次之后不能再修改，否则编译器会报错。
