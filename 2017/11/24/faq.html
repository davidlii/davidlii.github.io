2、NSString类型的属性为什么使用copy修饰符？

 需要注意的是，NSString 类型的属性并不是必须使用copy修饰，也可以使用 strong 等其他修饰符。那么什么时候要使用 copy 修饰呢？

 答：使用 NSMutableString 类型的源字符串（如mutStr）给 NSString 类型的属性（如str1）赋值时，如果不想 str1 的值因为 mutStr 的修改而意外的发生变化，则使用 copy 修饰。

 这是因为，copy 修饰的属性，在执行 setter 操作时，系统会默认拷贝一份新的对象并把属性的指针指向这个新的对象，新的对象的值跟源对象是一样的，但是二者内存地址不同。所以，后续在源对象上进行的修改，并不会影响到新拷贝的这个对象。

 而使用 strong 修饰的 NSString 类型的属性，其内存地址与源字符串一样，都指向源字符串在常量区的值。因此当源字符串的值发生变化时，属性会跟着变化。

 另外，当使用 NSString 类型的源字符串给 NSString 类型的属性赋值时，因为源字符串是不可变的，所以属性不论使用 copy 还是 strong 修饰，都是指向源对象所指向的内存地址（常量区的值）。

3、block为什么使用copy修饰？

 block 是一个对象, 所以 block 理论上是可以 retain/release 的。但是 block 在创建的时候它的内存是默认是分配在栈(stack)上，而不是堆(heap)上的。 所以它的作用域仅限创建时候的当前上下文(函数, 方法…)，当你在该作用域外调用该 block 时, 程序就会崩溃。

 一般情况下你不需要自行调用 copy 或者 retain 一个 block。 只有当你需要在 block 定义域以外的地方使用时才需要 copy。copy 会将 block 从内存栈区移到堆区。

 其实 block 使用 copy 是MRC留下来的。 在MRC下, 如上述, 在方法中的 block 创建在栈区, 使用 copy 就能把他放到堆区, 这样在作用域外调用该 block 程序就不会崩溃。

 但在ARC下, 使用 copy 与 strong 其实都一样，因为 block 的 retain 就是用 copy 来实现的。

5、指针&地址

 定义指针变量的语法：类型 变量名; (例如：NSString name = @”Dav”;)

 指针变量本质是保存了其指向的变量在内存中的地址。

 这里，*name是指针变量，指向@”Dav”。

 后者是存储在常量区的，其在内存中有个对应的地址，比如：0x16fdc9f0c。
